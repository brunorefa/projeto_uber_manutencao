# Projeto Uber Manutenção

## Descrição
Projeto em desenvolvimento para simular uma plataforma de solicitação de serviços de manutenção veicular sob demanda, inspirada no modelo de funcionamento de aplicativos como a Uber.

## Objetivo
Simular a integração entre usuários e prestadores de serviço, explorando conceitos de APIs, regras de negócio, alocação de recursos e fluxo de atendimento.

## Contexto de Uso (Simulação)
Em um cenário real, este sistema poderia ser utilizado por motoristas que enfrentam problemas mecânicos e precisam de atendimento rápido no local onde o veículo se encontra.

A plataforma atua intermediando a solicitação, localizando o prestador de serviço mais próximo e acompanhando o atendimento.

## Funcionalidades
- Abertura de solicitação de manutenção veicular
- Simulação de localização do veículo
- Alocação do prestador de serviço mais próximo
- Acompanhamento do status da solicitação
- Estrutura preparada para integração via API REST

## Tecnologias Utilizadas
- Linguagem de programação (conforme implementação)
- APIs REST
- Lógica de Negócio
- Simulação de Geolocalização
- Git / GitHub

## Exemplo de Fluxo
1. Usuário informa um problema no veículo
2. Sistema registra a localização
3. Plataforma identifica o prestador mais próximo
4. Solicitação é encaminhada
5. Status do atendimento é atualizado até a conclusão

## Como Executar
1. Clonar o repositório
2. Configurar o ambiente conforme a linguagem utilizada
3. Executar a aplicação
4. Simular solicitações de manutenção
